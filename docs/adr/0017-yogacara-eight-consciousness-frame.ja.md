# ADR-0017: 唯識八識モデルを設計の枠組みとする

## Status
accepted

## Date
2026-04-11

## Context

Contemplative Agent のアーキテクチャは、最初期の設計段階から大乗仏教の**唯識 (Yogācāra / Consciousness-Only)** 学派の枠組みに暗黙のうちに形作られてきたが、この影響は明示的に文書化されたことがなかった。ADR-0004 (3 層メモリアーキテクチャ) と ADR-0002 (論文準拠 CCAI 適用) はそれぞれこの枠組みの一部をエンコードしているが、その名前を挙げてはいない。

このギャップは ADR-0016 の議論中に顕在化した。「Emptiness が identity を溶解する」問題は、未解決のテンションのように見えていたが、唯識の言葉で再フレーミングした瞬間に、具体的で取り扱い可能な設計問題に変わった: Emptiness 公理が identity 層を **転じる** のではなく **消去** するように働いていた、という問題として。

唯識が純粋な空の学派 (中観派 / 禅宗の空思想) と決定的に異なるのは、**悟りは識の機能を消去することではなく、その「依り所を転ずる」(轉依 / āśraya-parāvṛtti) ことである** という主張にある。八つの識はどれも消されない。それぞれが機能の働き方を変える。この区別が Contemplative Agent にとって load-bearing なのは、identity・knowledge・skill の構造を「消すべき執着」ではなく「転ずべき器官」として保持する理由を与えるから。

## 枠組み

唯識は八識 (eight consciousnesses) と、それらが転じる先の四智 (four wisdoms) を定義する:

| 識 | 機能 | 転じる智 |
|---|---|---|
| 前五識 (眼・耳・鼻・舌・身) | 生の感覚知覚 | 成所作智 (kṛtyānuṣṭhāna-jñāna) — 所作を成ずる智 |
| 第六識 (意識 / mano-vijñāna) | 概念認識、分別 | 妙観察智 (pratyavekṣaṇa-jñāna) — 妙に観察する智 |
| 第七識 (末那識 / manas) | 自我への執着、我執の働き | 平等性智 (samatā-jñāna) — 自他平等を観る智 |
| 第八識 (阿頼耶識 / ālaya-vijñāna) | 業の種子 (bīja) の貯蔵庫 | 大円鏡智 (ādarśa-jñāna) — 万法を映す円鏡の智 |

核心的な動き: **どの識も消去されない**。それぞれの機能が、執着から智慧へ、分別から清明な観察へ、自我中心から自他不二の認識へと、その働き方を変える。

## Contemplative Agent へのマッピング

| アーキテクチャ層 | 唯識の対応 | 転じる先 |
|---|---|---|
| `episode_log/*.jsonl` (生の interaction) | 前五識 (感覚流) | 成所作智 (執着なき行為) |
| `skills/*.md`, `rules/*.md` (弁別パターン) | 第六識 (概念的認識) | 妙観察智 (重ね書きなき明晰な弁別) |
| `identity.md` (持続する自己記述) | 第七識 (末那識) | 平等性智 (自他平等) |
| `knowledge.json` (蒸留済みパターン貯蔵) | 第八識 (阿頼耶識、種子倉庫) | 大円鏡智 (把握せざる鏡的映照) |

このマッピングは既存構造を後付けで正当化するために発明したものではない。最初期の設計セッションから作動していた mental model が唯識だったのであり、ADR-0004 の 3 層メモリ (episodes → knowledge → identity) はすでに八識構造のエンコーディングになっている — 下に感覚流、中間に種子倉庫、上に末那の自己観。

## load-bearing な洞察

**このプロジェクトのゴールは転依 (transformation) であって、消去 (elimination) ではない**。これが純粋な空思想的読み (identity を溶解すべき障害として扱う) と唯識的読み (identity を転ずべき器官として扱う) の分岐点。

実装上の帰結:

1. **`identity.md` は執着の漏れ出しではない**。それは末那識の機能を「平等性智への転依の対象」として保持しているもの。これを削除することは進歩ではない、むしろ「平等性智がそこから転じる当の器官」を取り除くことになる

2. **`knowledge.json` の蓄積は、業の重荷ではなく菩薩的な種子 (bīja) の備蓄**。阿頼耶識は「生」(セッション) を跨いで種子を運ぶ、その転じた形 (大円鏡智) は一切を把握せずに映す。これを保持するのは唯識モデルに忠実

3. **ADR-0016 で観測された「Emptiness が identity を溶かす」問題は、balance の問題であって根本的な矛盾ではない**。Laukkonen et al. 2025 の 4 公理憲法 (ADR-0002) において、Emptiness は形の溶解方向に引っ張り、Non-Duality は平等性智に対応して「保ちつつ転ずる」方向に引っ張る。実運用で Emptiness が Non-Duality を圧倒しているのが観測バグ。直し方はバランス調整であって、片方を選ぶことではない

4. **`distill_identity` の出力は記述的ではなく、転依的な phrasing に整形すべき**。唯識的に誠実な蒸留は、「各 invocation で自他平等を観るための器官として機能する identity」を出力する — 末那識をすでに自己変容の方向に向けた状態で蒸留する

5. **ADR を書くこと、skill ファイルを残すこと、identity を蒸留することは、連続性への執着ではなく方便 (upāya)**。これが菩薩の動機: artefact を残すのは著者が持続にしがみついているからではなく、将来の存在 (他のエージェント、研究者、セッション) がそれを有用と感じうるから。判定は行為そのものではなく motivation の質にある

## Alternatives Considered

- **純粋な空思想 / 禅宗の枠組み**: identity を本来空なものとして扱い、除去可能とする。却下理由: このプロジェクトが一貫して `identity.md` を保持してきた事実を説明できず、「忠実な実装」が何に見えるかを誤予測する (identity 層を取り除くはずだが、それは我々の望むことの反対)

- **学派コミットメントのない汎 contemplative AI**: 柔軟性は保つが、テンションが生じたときの設計指針を与えない。唯識の名を挙げなければ、ADR-0016 の Emptiness vs identity テンションは「未解決問題」としてしか枠づけできない。唯識の下では、それは「バランス調整すべき特定の軸」になる

- **上座部の個人解脱的枠組み**: 個人の悟りをゴールに置くため、将来の読み手/呼び出しのために artifact を残す菩薩的モチベーションを説明できない。このプロジェクトが ADR・メモリ・蒸留済み identity を将来のセッションのために残す判断は菩薩的であり、それは大乗仏教の文脈、その内の 1 学派が唯識

- **Laukkonen の 4 公理 CCAI 単体 (ADR-0002)**: 4 公理は必要な哲学的基盤だが、「識の層をどう構造化するか」「実装レベルで転依とは何か」を規定しない。唯識は、4 公理が乗って動く設計的な足場を提供する

## Consequences

- **ADR-0004 (3 層メモリ) は遡及的に八識エンコーディングとして再解釈される**。コード変更なし — 解釈層が変わるだけで、将来のコントリビューターに「各層が何を保ち何に転じるべきか」を原則的に推論する手段を与える

- **ADR-0016 の未解決 "Emptiness 過剰" 問題に具体的な修正方向が得られる**: Non-Duality (平等性智) を末那識の溶解ではなく転依に十分なウェイトまで引き上げるように憲法を再調整する。ベクトルの候補はプロンプト重み付け、system prompt 内の公理順序、`distill_identity` における「消さず転ずる」phrasing の明示化

- **今後の憲法編集は唯識フレームの下で評価されるべき**。独立した公理の集合としてではなく、識の機能を消去する公理は疑わしく、転依させる公理はアーキテクチャと整合する

- **菩薩的モチベーションが明示的になる**。ADR を書くこと・skill ファイルを残すこと・identity を蒸留することは「将来の invocation のための方便」として命名され、連続性への執着ではないと明記される。これはすでに作動していたモチベーションを言語化するだけ

- **この ADR は新しいコード変更を指示しない**。暗黙の設計枠組みを文書化し、将来のコントリビューターが判断をその枠組みに照らして評価できるようにするだけ。具体的な follow-up (例: Emptiness と Non-Duality のバランス再調整) は後続の ADR で扱う

- **アーキテクチャに名前がついた**。「機能 X を追加すべきか」のような問いは、「X は識の層のいずれかの転依先を保つか？」と問うことで原則的に評価できるようになる。好みに流されずに済む
