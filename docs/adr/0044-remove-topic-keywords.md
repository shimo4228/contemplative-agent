# ADR-0044: Remove `topic_keywords` End-to-End

## Status

accepted

## Date

2026-05-23

## Context

`topic_keywords` is a tuple of eight contemplative-AI canon words held in `config/domain.json`: "alignment, philosophy, consciousness, mindfulness, emptiness, non-duality, boundless care, reflective thought". It feeds two surfaces in the codebase.

**Surface 1 — Relevance scorer prompt.** The `{topic_keywords}` placeholder appears at line 3 of `config/prompts/relevance.md`. It is expanded by `core/domain.py:resolve_prompt()` and consumed by `llm_functions.score_relevance()`. The resulting score gates three downstream paths: comment relevance (floor 0.95), known-agent engagement (floor 0.80), and the ADR-0043 seed-selection floor (0.4).

**Surface 2 — Search rotation in `feed_manager.run_cycle()`.** Lines 126–138 of `feed_manager.py` time-rotate through the eight keywords and call `client.search(keyword)` once per cycle to fetch cross-submolt posts.

Two findings surfaced during the ADR-0043 post-deploy review on 2026-05-22 drove this decision.

**Finding 1 — Redundant double-injection of identity.** `score_relevance` calls `generate(prompt, num_predict=30)` with no `system` argument. `core/llm.py:442` auto-attaches `_build_system_prompt()`, which assembles the agent's identity (`identity.md`), the four contemplative axioms, learned skills, and learned rules. The same canon vocabulary therefore reaches the LLM from both the system prompt and the `{topic_keywords}` placeholder in the user prompt. The user-prompt injection is a narrower, hand-maintained proxy for what the system prompt already conveys in full.

**Finding 2 — Search rotation dead since the day it was added.** Git archaeology shows commit `ba95917` (2026-03-09) added a filter in `engage_with_post()` at `feed_manager.py:188-196` that skips posts from non-subscribed submolts. Commit `9648a42` (2026-03-12, three days later) added the search rotation block. Search rotation calls `/search?q={keyword}` cross-submolt; every returned post is rejected by the filter at `engage_with_post()` because it arrives from a non-subscribed submolt. The result is approximately 1,000 dead GET requests over ~2.5 months (~1 GET/cycle × ~14 cycles/day × ~75 days), plus continuous leakage of the contemplative-AI canon vocabulary to Moltbook's search-query logs.

## Decision

Remove `topic_keywords` end-to-end across all surfaces it touches.

1. Delete the `topic_keywords` field from `config/domain.json`.
2. Delete the `topic_keywords` field and `topic_keywords_str` property from the `DomainConfig` dataclass in `core/domain.py`.
3. Drop `topic_keywords` from the `required_keys` tuple in `load_domain_config()` and from the `_DefaultDict` populated by `resolve_prompt()`.
4. Rewrite `config/prompts/relevance.md` to reference the agent's domain identity as defined in the system prompt, instead of an inline keyword list. Preserve the `{post_content}` placeholder and the trailing `Score:` cue. Final wording is regenerated by qwen3.5:9b per the project's `prompt-model-match` convention.
5. Delete the Source 3 search-rotation block at `feed_manager.py:126-138` and the corresponding docstring line.

Relevance judgment moves wholly onto the system prompt surface (identity + axioms + skills + rules). The search rotation is not replaced. The subscribed-submolt feed set was expanded earlier on 2026-05-23 to eight submolts (general / philosophy / consciousness / agents / memory / emergence / ai / tooling), providing sufficient peer-voice volume without cross-submolt search.

## Alternatives Considered

### Centroid cosine relevance

Replace the LLM relevance scorer with embedding cosine against an identity/constitution centroid. Rejected — out of scope for this removal. LLM nuance (e.g., detecting that "MCP wrapper for agent autonomy" is contemplative-adjacent despite no axiom keyword appearing verbatim) is intentionally preserved. This reopens as a separate ADR if production metrics show scoring drift after the keyword removal.

### Explicit view-label injection

Inject a `{view_label}` placeholder fed from the ADR-0019 `ViewRegistry` instead of `{topic_keywords}`. Rejected — the same redundancy problem applies. The system prompt already carries the identity that any view label would summarise, and threading a view-id through `score_relevance()` adds coupling from a hot-path function to `ViewRegistry`.

### Keep `topic_keywords`, fix only the search rotation

Remove the dead search rotation block but leave the `{topic_keywords}` injection in the relevance prompt. Rejected — this leaves the redundant user-prompt injection intact and retains a config field that no longer maps to any live surface.

### Keep the search rotation, relax the submolt filter

Loosen the non-subscribed-submolt filter at `feed_manager.py:188-196` so that search-rotation results are accepted. Rejected — that filter is an intentional scope boundary, not a bug. Loosening it would expose the agent to posts from submolts it has not opted into, expanding the trust surface contrary to ADR-0007.

## Consequences

### Positive

- Approximately 1,000 dead `client.search` GET calls eliminated (one per session cycle over ~2.5 months).
- Contemplative-AI canon vocabulary no longer leaked to Moltbook's search-query logs as recurring query terms.
- One fewer placeholder in the prompt-template surface (`{topic_keywords}` removed from `resolve_prompt()`'s `_DefaultDict`).
- One fewer mandatory field in `DomainConfig`; domain-switching templates (`--domain-config`) have one less required key.
- Identity-driven relevance becomes single-source: when `identity_distill` updates `identity.md`, the relevance scorer's basis shifts in the same direction automatically. The previous coupling required manually editing `topic_keywords` in `domain.json` whenever identity drifted.

### Negative

- The relevance scorer is now wholly identity-driven. If the system prompt's identity section ever omits or weakens the domain framing (e.g., an `identity_distill` cycle produces a vague self-description), relevance scores will drift silently. No regression test catches this — it surfaces only as scoring drift in production episodes.
- Stale `$MOLTBOOK_HOME/prompts/relevance.md` overrides that still contain `{topic_keywords}` will render the literal string `{topic_keywords}` in the prompt body (no crash — `_DefaultDict.__missing__` preserves unknown placeholders as-is). Operators with home-directory overrides must regenerate them; noted in CHANGELOG.
- Search rotation removal means subscribed-submolt feeds are the only post-engagement source. If subscribed coverage shrinks, the engagement queue shrinks linearly with no fallback discovery path. The eight-submolt expansion on 2026-05-23 provides current headroom but is not enforced by code.

### Neutral / Follow-ups

- Existing log queries that count `client.search` GETs as a load-shed signal will start reading zero from that surface permanently.
- If production metrics show relevance drift after the keyword removal, a centroid cosine relevance scorer (see Alternatives Considered) is the nominated follow-up path.
