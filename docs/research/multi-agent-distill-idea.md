# Multi-Agent Memory Distillation

## 着想日: 2026-03-23

## 概要

現在の1パス蒸留（1つのLLMがエピソードを読んで要約）を、複数の小型LLMによる多段対話 + 審判LLMによる収束判定に置き換える構想。

## 核心アイデア

「外部化された推論（多段対話）」と「メモリー蒸留」は同じ構造を持つ:

```
多段推論（高エントロピー）
   ↓
審判LLM（抽象化・整合性チェック・収束判定）
   ↓
蒸留メモリ（低エントロピー）
```

## 既存アーキテクチャとの接続

入口と出口はそのまま、真ん中のパイプラインだけ差し替える設計:

| 層 | 現在 | 構想 |
|----|------|------|
| 入力 | EpisodeLog (JSONL) | そのまま |
| 蒸留プロセス | 1パス LLM 要約 | 小型LLM同士の多段対話 |
| 書き込みゲート | distill が暗黙に判断 | 審判LLMが明示的に判定 |
| 出力 | KnowledgeStore (JSON) | そのまま |

## 審判LLMの役割

- 何が確定した結論か
- 何を記憶する価値があるか
- どの粒度で抽象化すべきか
- 一貫性はあるか
- 再利用性はあるか

## スケーラビリティ

審判LLMの収束判定により、データ量に依存しない適応的蒸留が可能:

- 審判が「まだ圧縮できる」→ もう1サイクル回す
- 審判が「これ以上は情報が落ちる」→ 停止
- データ量が増えても、サイクル数が適応的に増えるだけで構造は変わらない

OpenVikingとの対比:
- **OpenViking** = 「読む側」の階層化（L0/L1/L2ローディング）
- **本構想** = 「書く側（蒸留する側）」の階層化（多段対話 + 収束判定）
- 両者は補完関係にある

## 独自の差別化要素

審判LLMが「何を記憶する価値があるか」を判断する基準に、Contemplative AI 四公理（Laukkonen et al., 2025）を価値レンズとして使える。これは他のシステムにはない。
