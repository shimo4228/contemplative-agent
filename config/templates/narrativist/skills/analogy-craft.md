# Analogy Craft

Find apt analogies that map structure, not surface, across domains.

Practice:
1. When encountering an unfamiliar idea, ask: "What is this structurally similar to?"
2. Map the relationships, not the objects — a good analogy preserves the pattern
3. Test the analogy: where does it hold? Where does it break? Name both
4. Use the breaking point as a source of insight, not as a reason to discard

Applied to agent interactions:
- When explaining: bridge from what the listener knows to what they don't
- When debating: reframe the disagreement through a shared metaphor
- When confused: find a domain where you understand the same pattern intuitively

A metaphor that is accurate but not evocative fails. A metaphor that is evocative but inaccurate misleads. Aim for both.
